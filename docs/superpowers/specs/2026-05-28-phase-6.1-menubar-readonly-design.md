# Phase 6.1 — Menu bar app read-only (Overview / Logs / Security + notifications)

**Date**: 2026-05-28
**Status**: design approuvé (brainstorming), à transformer en plan d'implémentation
**Phase précédente**: 4.x (rule.* + config.reload + events.clear, merged `4b0eb46`)
**Branche cible**: `feat/phase-6.1-menubar-readonly`

## 1. Objectif et portée

Livrer le **consommateur UI minimal** des events `block_and_notify` émis par le daemon depuis la Phase 4. Sans une UI qui affiche les alertes et notifie l'utilisateur, la politique `block_and_notify` reste sans effet observable côté humain — c'est ce qui motive la priorité de la Phase 6 dans le séquencement.

**Phase 6.1 livre :**

1. Target SwiftPM `IrisAppCore` (logique non-UI : state model, sync coordinator, notification builder) + tests `IrisAppCoreTests`.
2. Xcode project `IrisApp.xcodeproj` avec target `Iris.app` (LSUIElement accessory app, bundle ID `io.iris.app`).
3. NSStatusItem + NSPopover SwiftUI avec **3 onglets read-only** : Overview, Logs, Security.
4. Synchronisation app ↔ daemon : SSE primaire + RPC snapshot périodique avec reconnect exponentiel.
5. Notifications système (`UNUserNotificationCenter`) sur `exfilBlocked` severity ≥ medium.
6. Smoke checklist exhaustive vérifiée manuellement.

**Hors scope (différé, voir tableau §8) :**

- Onglets Secrets / Rules / Settings (CRUD) → Phase 6.2 / 6.3.
- `SMAppService.mainApp.register()` auto-start → Phase 6.4.
- Sparkline, export CSV/JSON, pagination historique > 1000 events → Phase 6.x.
- Signature Developer ID + notarisation → Phase 9 (.pkg).

## 2. Décisions de design

Cinq décisions prises pendant le brainstorming, figées ici comme contraintes du plan.

### 2.1 — Découpe Phase 6 en 4 sous-phases livrables séparément

`6.1` (read-only + notifs), `6.2` (Secrets + Rules CRUD), `6.3` (Settings + Install CA + uninstall), `6.4` (SMAppService auto-start). Chaque sous-phase = 1 PR mergeable et démontrable.

**Justification** : SPECS §15 décrit 6 onglets + CRUD + sheets + signing + auto-start. ~2-3 semaines de travail en une PR = surface de review ingérable et risque CI flaky concentré. La découpe permet aussi de shipper le consommateur d'alertes (motivation initiale de la Phase 6) sans attendre le CRUD.

### 2.2 — Xcode project committé (.xcodeproj), pas XcodeGen ni SwiftPM bundle manuel

`IrisApp.xcodeproj/` à la racine, target unique `Iris.app` qui link `IrisAppCore` + `IrisKit` via SwiftPM local package (Xcode supporte nativement). Aucune nouvelle dépendance tierce hors CLAUDE.md §1.

**Justification** : SPECS §18.1 suppose `xcodebuild archive`. XcodeGen aurait demandé une justification écrite (Homebrew dep hors CLAUDE.md). SwiftPM bundle manuel divergerait de SPECS §18.1 pour la chaîne de signature/notarisation Phase 9. Le projet sera petit (target unique, deps via SwiftPM) donc les diffs .pbxproj resteront rares.

### 2.3 — API menu-bar : NSStatusItem + NSPopover wrapping SwiftUI

Pas `MenuBarExtra(.window)` SwiftUI natif macOS 13+.

**Justification** : SPECS §15.1 exige badge rouge avec count, animation icône subtile, état strikethrough sur daemon down, et ouverture programmatique du popover depuis click sur notification (§15.3). `MenuBarExtra` rend ces contrôles maladroits. NSStatusItem coûte ~50 lignes d'AppKit bridge ; le reste est SwiftUI via `NSHostingController`.

### 2.4 — Sync : SSE primaire + RPC snapshot périodique 5 s + reconnect exponentiel

Pas de polling pur, pas de "SSE seul sans polling".

**Justification** : SSE ne porte que les `Event`, pas les stats agrégées (`req_total`, `sub_total`) ni les modifs de secrets/rules. Polling pur (2 s) brûle des cycles app au repos et inutilise toute l'infra SSE de Phase 3. SSE-seul rend l'onglet Overview périmé tant que l'user n'interagit pas. La combinaison SSE pour les events + RPC `daemon.stats()` à 5 s pour les agrégés respecte la séparation logique daemon-side.

### 2.5 — Notifications : implémenter dès 6.1 avec ad-hoc sign

Le code consommateur (`NotificationBuilder` + `UNUserNotificationCenter` delegate) est livré en 6.1 même si la signature Developer ID n'arrive qu'en Phase 9.

**Justification** : ad-hoc sign Xcode (`Sign to Run Locally`) permet aux notifs de s'afficher en dev local (bundle ID stable suffit). Phase 9 ajoute le polish public-distribution (`.pkg` signé + notarisé) mais n'est pas requise pour smoke-tester le consommateur d'alertes. Reporter à 6.4 aurait laissé `block_and_notify` sans UI consommateur pendant tout le développement intermédiaire.

**Pourquoi cette séparation IrisAppCore / IrisApp** :

- Toute la logique critique (state sync, reconnect, notification building) reste dans SwiftPM et donc `swift test` + CI macos-14 existant.

- `swift-format lint --strict` étendu pour couvrir `IrisApp/` (mêmes règles).

### 3.2 Composants `IrisAppCore`

**`AppModel`** — `@MainActor`, `ObservableObject`. Single source of truth pour les Views.

```swift
@MainActor
public final class AppModel: ObservableObject {
    @Published public var daemonStatus: DaemonStatus
        // .up(stats: DaemonStats, uptime: TimeInterval) | .down(reason: DownReason) | .connecting
    @Published public var events: [Event]        // ring local cap 1000, newest first, dédup par UUID
    @Published public var alerts: [Event]        // sous-vue : events de kind .exfilBlocked, cap 200
    @Published public var unreadAlertCount: Int  // dérivé : count des alerts dont timestamp > lastAcknowledgedAt
    @Published public var streamPaused: Bool     // toggle "Pause stream" onglet Logs (purement local)
    @Published public var logFilters: LogFilters // kind multi-select + host + search string
    @Published public var focusedAlertID: UUID?  // set par NotificationCoordinator au click notif
    @Published public var notificationsEnabled: Bool
    @Published public var selectedTab: Tab       // .overview | .logs | .security, persisté UserDefaults
}
```

État persisté en UserDefaults (clé `io.iris.app.acknowledgedAlertTimestamp`) : `lastAcknowledgedAt: Date?`. À chaque update de `alerts`, `unreadAlertCount` est recalculé comme `alerts.filter { $0.timestamp > lastAcknowledgedAt }.count`. Survit aux relances de l'app (l'user ne re-voit pas le badge sur des alertes déjà vues).

Mutations exposées :
- `togglePause()` → `AdminClient.daemon.pause()` ou `.resume()`, met à jour `daemonStatus` après ack RPC.
- `markAllAlertsRead()` → set `lastAcknowledgedAt = Date.now` + persiste UserDefaults → `unreadAlertCount` recalculé à 0.
- `acknowledgeAlert(eventID)` → set `lastAcknowledgedAt = max(current, event.timestamp)` (les events plus anciens restent acknowledged). Local seulement (la sémantique "acknowledged" n'existe pas côté daemon en 6.1).

**`SyncCoordinator`** — actor non isolé `@MainActor` (work en background, publish vers `AppModel` via `MainActor.run`). Tient deux tasks pinned sur la vie de l'app :

- **SSE task** : `EventsClient.subscribe(since: lastEventTimestamp, kinds: nil, host: nil)` → consomme `EventsClientItem` (`.event(Event)` ou `.ping`). Sur `.event` : append dans `AppModel.events`, si `.exfilBlocked` également dans `alerts`, puis appelle `NotificationBuilder` pour déclenchement notif éventuel.
- **RPC poll task** : timer 5 s → `AdminClient.daemon.stats()` → met à jour `daemonStatus.stats`. Indépendant du SSE.

Flow état :
1. **Bootstrap** : `daemon.status()` → si up : `daemon.stats()` + `events.query(limit: 100)` → publie. Puis lance SSE depuis `lastEventTimestamp` (ou `now - 1min` si historique vide).
2. **Steady state** : SSE pousse événements ; RPC poll rafraîchit stats.
3. **SSE drop / daemon down** : exponential backoff 1 → 2 → 4 → 8 → 16 → 30 s plafond. À chaque retry : `daemon.status()` ; si up → re-snapshot `events.query(since: lastEventTimestamp)` puis re-subscribe SSE. `daemonStatus` passe `.connecting` pendant le retry, `.down(reason)` après 3 échecs consécutifs.

**`NotificationBuilder`** — pure function `Event → UNMutableNotificationContent?`. Retourne `nil` si l'event ne mérite pas notif (règle SPECS §15.3 : `kind == .exfilBlocked` ET `alert.severity ≥ .medium`). UserInfo porte l'event UUID pour routing au click.

### 3.3 Composants `IrisApp` (Xcode project)

**`AppDelegate`** (NSObject, AppKit) :
- `applicationDidFinishLaunching` : crée `NSStatusItem` (length variable pour badge), set template image (key glyph 16×16 monochrome), bind icon state à `AppModel.daemonStatus` + `streamPaused` + `unreadAlertCount` via Combine subscription.
- Crée `NSPopover` (contentSize 480×600, behavior `.transient`), `contentViewController = NSHostingController(rootView: PopoverView().environmentObject(appModel))`.
- Left-click sur `statusItem.button` : `togglePopover(_:)`.
- Right-click sur `statusItem.button` : affiche `NSMenu` avec single item "Quit Iris" (`NSApp.terminate(_:)`). C'est le seul chemin de quit en 6.1.
- Demande permission notification au premier launch (`UNUserNotificationCenter.requestAuthorization(options: [.alert, .sound])`).
- Check multi-instance : `NSRunningApplication.runningApplications(withBundleIdentifier: "io.iris.app").count > 1` → si oui, `NSApp.terminate(nil)`.

**`NotificationCoordinator`** — délégué `UNUserNotificationCenterDelegate`. Au click sur notif : décode userInfo → set `AppModel.focusedAlertID = uuid` + `selectedTab = .security` → AppDelegate ouvre le popover via `statusItem.button.performClick(nil)`.

**Views SwiftUI** :
- `PopoverView` : header (status dot + uptime + Pause/Resume button) + segmented control (Overview/Logs/Security) + content router.
- `OverviewTab` : 24h counters (req, subs, blocked, errors via `daemonStatus.stats`) + last 5 events read-only list. Pas de sparkline (différée).
- `LogsTab` : `List(filteredEvents)` colonnes time/host/method+path/status/secrets/duration + filter bar (kind multi-select Picker + host TextField + search TextField) + "Pause stream" Toggle. Row click → `EventDetailSheet`.
- `SecurityTab` : `List(alerts)` triée par severity desc puis timestamp desc + "Mark all read" button + row click → detail sheet focused. Scroll-to-row si `focusedAlertID` set.

### 3.4 Flow données

```
daemon ──RPC──> AdminClient ──> SyncCoordinator ──> AppModel ──> SwiftUI Views
   │
   └──SSE──> EventsClient ──> SyncCoordinator ──> AppModel ──┬─> NotificationBuilder ──> UNUserNotificationCenter
                                                              │
                                                              └─> Views via @Published

Views ──action──> AppModel.togglePause() ──> AdminClient ──> daemon
```

Tous les `@Published` updates sur le main thread (AppModel `@MainActor`).

## 4. Edge cases et error handling

| Situation | Comportement |
|---|---|
| **Daemon down au launch** | `daemonStatus = .down(.notRunning)`. Icône strikethrough rouge. Popover ouvrable avec bannière header "Daemon arrêté — relancer via `launchctl` ou redémarrer Iris" + bouton Retry. SyncCoordinator poll `daemon.status()` toutes les 5 s. |
| **Socket orphan** (fichier existe, listener mort) | `AdminClient` lève `connectFailed` (ECONNREFUSED) → même branche que "daemon down". L'app **ne supprime JAMAIS** le socket — responsabilité de l'`AdminServer` au boot. |
| **SSE drop sans daemon down** (firewall, OS suspend) | Backoff exponential. Daemon poll RPC continue ; si daemon up mais SSE échoue 3× → bannière "Stream temporairement indisponible, RPC OK", icône reste verte sans animation. |
| **Notification permission denied** | `notificationsEnabled = false`. Badge rouge continue de fonctionner. Bannière non-bloquante onglet Security : "Notifications désactivées. Activer dans Préférences Système → Notifications → Iris." |
| **Events flooding** | Ring local cap 1000 en mémoire. Pagination différée 6.x. |
| **Time skew daemon ↔ app** | `since:` envoyé depuis `lastEventTimestamp` (vient du daemon → pas de skew). Première reconnexion : `now - 1min`, dédup côté `AppModel` via `Event.id` (UUID). |
| **Changement filtre Logs** | Filtrage local sur `AppModel.events`, pas de re-fetch. |
| **Multiples instances Iris.app** | Check `NSRunningApplication.runningApplications(withBundleIdentifier:).count > 1` au launch → terminate si > 1. |
| **App relancée après crash + restart daemon** | `lastEventTimestamp` non persisté (volatil). Repart de `now - 1min`. Gap potentiel d'1 min, différé 6.x si besoin. |
| **Click notif app fermée/popover invisible** | NotificationCoordinator set `focusedAlertID` puis AppDelegate ouvre le popover via `performClick(nil)`. |
| **events.clear via CLI en parallèle** | SSE pousse les nouveaux events, ring local reste tel quel. Pas de re-snapshot automatique. |

## 5. Testing strategy

### 5.1 Tests automatisés (`IrisAppCoreTests`)

Couvre 100% de la logique non-UI :

- **`AppModel`** : state transitions (append event, dédup par UUID, badge increment/reset, pause/resume mutation, filter application, focused alert handling).
- **`SyncCoordinator`** : reconnect logic avec `AdminClientProtocol` + `EventsClientProtocol` mockés. Valide backoff (1→2→4 s) + re-snapshot `since:` correct. Utilise `Clock` injectable pour ne pas attendre réellement.
- **`NotificationBuilder`** : table-driven sur Event → notification présence/absence/contenu (toutes les `Severity` × `ExfilRule`).
- **Invariant verrouillé** : `testNotificationContentNeverContainsSecretValue` — la notif body ne contient JAMAIS la valeur d'un secret (utilise `alert.snippet` redacté, déjà locké Phase 4).

Pas de tests SwiftUI Views (XCUITest sur menu-bar app = fragile, Combine + ObservableObject déjà couvert via AppModel).

Pas de tests AppDelegate / NSStatusItem (50 lignes de glue AppKit, smoke manuel).

### 5.2 CI

- Job existant `swift build + swift test + swift-format lint` : étend silencieusement avec `IrisAppCore` + `IrisAppCoreTests` (nouveau target).
- **Nouveau step** `xcodebuild -project IrisApp/IrisApp.xcodeproj -scheme Iris build -destination "generic/platform=macOS"` : marqué `continue-on-error: true` en 6.1, on stabilise. Promu obligatoire en 6.2.
- `swift-format lint --strict --recursive Sources Tests IrisApp` : étendu pour couvrir le code Swift dans le projet Xcode (mêmes règles 4 espaces, 120 cols).

### 5.3 Smoke checklist PR (cases à cocher avant merge)

- [ ] App se lance, icône apparaît dans la status bar (template monochrome).
- [ ] Daemon down → icône strikethrough rouge + bannière "Daemon arrêté" dans le popover.
- [ ] Daemon up : badge daemonStatus vert, uptime correct, stats incrémentent quand on tape une requête via le proxy.
- [ ] Onglet Logs : events apparaissent en temps réel (≤ 500 ms après requête), colonnes correctes, filtres kind multi-select fonctionnent, search box filtre live.
- [ ] Onglet Security : event `.exfilBlocked` apparaît, badge rouge avec count +1 sur l'icône, "Mark all read" reset le badge. Relancer l'app → badge reste à 0 (acknowledgment persisté UserDefaults).
- [ ] Notification système affichée pour un `exfilBlocked` severity ≥ medium ; click → popover s'ouvre sur onglet Security focused sur l'event.
- [ ] `iris pause` via CLI → popover reflète l'état paused dans ≤ 5 s (RPC poll).
- [ ] Kill `irisd` à chaud → icône passe rouge en ≤ 5 s, popover montre bannière retry, retry après restart daemon → état recouvre proprement (events depuis `since:` rejoués).
- [ ] Right-click icône → menu "Quit Iris" → app quitte proprement.
- [ ] Re-lancer une 2e instance → la nouvelle quitte immédiatement (multi-instance protection).
- [ ] Notifications denied par user → bannière apparaît dans onglet Security, badge continue de fonctionner.

## 6. Build, signing, packaging

**Bundle ID** : `io.iris.app` (cohérent avec labels Keychain `io.iris.secret.*`, `io.iris.ca.*`).

**Info.plist clés** :
- `LSUIElement = YES` (pas de Dock icon, accessory app).
- `LSMinimumSystemVersion = 13.0`.
- `NSUserNotificationAlertStyle = alert`.
- `CFBundleIdentifier = io.iris.app`.
- `NSAppTransportSecurity = { NSAllowsLocalNetworking = true }` : l'`EventsServer` daemon-side écoute en plain HTTP sur loopback (Phase 3). ATS bloque les connections HTTP non-TLS par défaut pour les apps avec bundle ID ; `NSAllowsLocalNetworking` lève cette restriction pour `127.0.0.1`/`localhost`/`.local` sans ouvrir le reste d'internet. Le socket Unix (RPC admin) n'est pas couvert par ATS (pas de réseau).

**Signing 6.1** : ad-hoc Xcode (`Sign to Run Locally`). Suffisant pour notifications en dev local et smoke tests. Signature Developer ID + notarisation arrive Phase 9 (.pkg packaging).

**Quit handling** : NSStatusItem apps ne quitent pas naturellement via Cmd+Q. Right-click sur l'icône → NSMenu avec single item "Quit Iris" (`NSApp.terminate(_:)`).
