# Phase 6.3b — Onglet « Settings » de l'app menu-bar (UI)

> Design validé le 2026-06-05. Suite directe de la **Phase 6.3a** (modèle de config
> unifié `config.json` app-first, déjà mergée — squash `837ba52`, PR #42), dont le
> §11 capturait ce design UI. La 6.3b est **l'UI qui consomme 6.3a** : elle n'ajoute
> qu'un seul verbe RPC (`config.path`, cf §5) ; tout le reste est du câblage de RPC
> existants. Agencement visuel retenu en brainstorming : **C — `Form` natif macOS à
> `GroupBox` encadrés**.

## 1. Objectif

Ajouter le **6e onglet** `Settings` au popover (`IrisApp/IrisApp/PopoverView.swift`,
qui liste aujourd'hui 5 onglets, `TabBar.items:147-153`) pour :
- **Afficher** les réglages du daemon (`config.get`) ;
- **Éditer sans friction** (G7 SPECS) les champs *hot* via `config.set` ;
- **Gérer la CA** : statut « trust store » (`ca.is_trusted`) + Install/Uninstall
  (`CATrustStore.install/uninstall`, panneau d'auth présenté par l'app signée
  Developer ID — 8a/8b mergées) ;
- **Révéler** `config.json` dans le Finder + **Reload** (`config.reload`).

## 2. Décisions de cadrage (verrouillées)

| Décision | Choix | Justification |
|----------|-------|---------------|
| Agencement | **C — `Form` + `GroupBox` par domaine** | Idiome macOS canonique pour des réglages (« Réglages Système »). Un panneau de réglages est légitimement différent des onglets-listes ; on assume l'introduction de `Form`/`GroupBox` (absents ailleurs dans l'app). |
| Périmètre destructif | **CA uninstall seul** | Pas de purge Keychain (les secrets restent supprimables un par un dans l'onglet Secrets). Le retrait LaunchAgent est reporté à 6.4/Phase 7 ; introduire une purge irréversible à demi-histoire serait asymétrique. Décision utilisateur 2026-06-05. |
| Quit | **Non dupliqué** | Déjà présent via clic-droit sur l'icône (`AppDelegate.swift:163`, `showQuitMenu`). |
| **D1 — Application de `config.set`** | **Immédiate au commit** | Idiome Réglages Système + objectif sans friction : changement de `Picker` ⇒ apply ; valeur numérique ⇒ apply **au commit** (Entrée/perte de focus ou trailing-edge debouncé), **pas à chaque incrément** (sinon spam RPC). Pas de bouton « Apply » global. |
| **D2 — Source du chemin `config.json`** | **Nouveau RPC `config.path`** | `ConfigStore.path` est `private` (`ConfigStore.swift:19`) et aucun RPC ne l'expose. Le RPC (symétrique à `ca.export_path`) fait du daemon la source de vérité, robuste à un `--config <path>` override. Dériver côté client serait faux sous override. |
| **D3 — Seam CA install** | **Protocole `CATrustInstalling` dans IrisAppCore** | `CATrustStore.install` shell-out + GUI = non testable. Un seam injectable rend `AppModel.installCA()/uninstallCA()` unit-testables avec un mock ; l'impl par défaut wrappe `CATrustStore`. Le shell-out réel + panneau = smoke. |
| Champs structurels | **Lecture seule** | `broker.*` (listen, ports, socket, log_level, retention, ring) affichés grisés. Conséquence : tous les champs **éditables** étant *hot*, `requires_restart` n'est **jamais** déclenché par l'UI en 6.3b → pas de bannière « redémarrage requis » à construire. |
| `log_level` | **Lecture seule** | Le câblage du log level au backend logging reste un TODO connu (hors-scope, → Phase 5.3.1.x). Affiché, non éditable. |

## 3. Contenu de l'onglet (agencement C)

`Form` vertical (dans un `ScrollView` si nécessaire, popover 480×600) avec un
`GroupBox` par domaine :

1. **Security** (éditable)
   - `on_exfil_attempt` : `Picker` sur `ExfilAttemptPolicy.allCases`
     (`.blockOnly` / `.blockAndNotify` / `.blockNotifyPause`,
     `Config.swift:165-168`, `String, CaseIterable`).
   - `max_substitutions_per_minute` : `Int`, édité via `TextField` numérique
     (+ `Stepper` d'incrément), **appliqué au commit** (D1 : `onSubmit`/perte de
     focus ou debounce, pas à chaque incrément).
2. **Backups** (éditable)
   - `backups.max_count` : `Int` ≥ 1 (`BackupsConfig.maxCount`, `Config.swift:59-63`),
     même mécanique d'apply-au-commit que ci-dessus.
3. **Certificate Authority**
   - Pastille de statut depuis `caTrusted` (Trusted / Not trusted / inconnu).
   - Bouton contextuel : **Install** si non trusted, **Uninstall** si trusted.
4. **Connection** (lecture seule, grisé) — depuis `broker` (`Config.swift:99-113`) :
   `listen`, `events_listen`, `admin_socket`, `log_level`,
   `event_retention_days`, `event_ring_size`.
5. **Footer** : `Reveal config.json` (révèle dans le Finder) · `Reload`
   (`config.reload`).

## 4. Architecture & composants

### Vue — `IrisApp` (smoke-only)
- **Nouveau** `IrisApp/IrisApp/SettingsTab.swift` : le `Form`/`GroupBox` ci-dessus.
  `@EnvironmentObject var model: AppModel`, `let admin: AdminCalling` (même forme
  que `SecretsTab`/`RulesTab`). Édition immédiate (D1) : `onChange`/`onSubmit` →
  `model.setConfig(...)`. `.onAppear` → `model.loadConfig()` + `model.refreshCATrust()`
  (RPC bon marché ; reflète un `iris ca install` ou `iris config set` survenu hors-bande).
- **`PopoverView.swift`** : ajout `case .settings: SettingsTab(admin: admin)` au
  `switch` (`PopoverView.swift:21-27`) et d'une `Item(tab: .settings, title:
  "Settings", symbol: "gearshape")` dans `TabBar.items` (`PopoverView.swift:147-153`).

### Cœur — `IrisAppCore` (testé via `swift test`)
- **`AdminCalling`** (`Sources/IrisAppCore/Protocols/AdminCalling.swift`) étendu :
  - `func fetchConfig() async throws -> Config`
  - `func setConfig(updates: [ConfigSetParams.Update]) async throws -> ConfigSetResult`
  - `func reloadConfig() async throws -> ConfigReloadResult`
  - `func configPath() async throws -> String`  *(nouveau RPC, §5)*
  - `func isCATrusted() async throws -> Bool`
  - `func caExportPath() async throws -> String`
  Implémentés dans le conformer IrisKit (`AdminClient`) en mappant
  `config.get/set/reload/path`, `ca.is_trusted`, `ca.export_path`.
- **`AppModel`** : nouvel état publié `config: Config?`, `caTrusted: Bool?` +
  mutations (pattern « RPC puis re-fetch », comme 6.2) :
  - `loadConfig()` → `fetchConfig()` ;
  - `setConfig(_ updates:)` → `setConfig(updates:)` puis re-fetch `config` ;
  - `reloadConfig()` → `reloadConfig()` puis re-fetch ;
  - `refreshCATrust()` → `isCATrusted()` ;
  - `installCA()` / `uninstallCA()` (cf seam ci-dessous).
- **Seam CA (D3)** : `protocol CATrustInstalling: Sendable { func install(pemPath:
  String) throws ; func uninstall(pemPath: String) throws }` + impl par défaut
  `SystemCATrustInstaller` wrappant `CATrustStore.install/uninstall`
  (`CATrustStore.swift:63,69`). `AppModel` reçoit un `caInstaller: CATrustInstalling`
  (défaut `SystemCATrustInstaller()`). `installCA()` : si `caTrusted == true` → no-op
  (idempotent, comme le CLI `CACommands.swift:107`) ; sinon `path = try await
  admin.caExportPath()` → `caInstaller.install(pemPath: path)` **hors du main actor**
  (le panneau d'auth bloque jusqu'à réponse — ne pas geler le popover) → `await
  refreshCATrust()`. `uninstallCA()` symétrique.
- **Mapping testable** : la conversion contrôle → `[ConfigSetParams.Update]`
  (ex. `Picker` `.blockOnly` ⇒ `{key:"security.on_exfil_attempt", value:"block_only"}`)
  est une fonction pure (réutilise `ExfilAttemptPolicy.rawValue`), testable sans UI.

### Bootstrap
`AppModel` charge aujourd'hui status+events+secrets+rules. Ajout de `fetchConfig` +
`isCATrusted` au bootstrap → **casse le test d'assertion exacte des calls** (même
effet qu'en 6.2 avec `listSecrets`/`listRules`) → mettre à jour le test attendu
(`+fetchConfig`, `+isCATrusted`).

## 5. Backend ajouté : RPC `config.path` (D2)

Seul changement daemon de la 6.3b.
- `AdminMethod` : nouveau cas `configPath = "config.path"`
  (`Sources/IrisKit/IPC/AdminProtocol.swift`).
- Résultat `ConfigPathResult { path: String }` (Codable/Sendable/Equatable).
- `ConfigStore` : exposer le chemin — `public var filePath: String { path.path }`
  (`ConfigStore.swift:19` est `private`).
- `AdminDispatcher` : router `.configPath` vers le store (symétrique à `.caExportPath`,
  `AdminDispatcher.swift:204-206`). Le dispatcher accède au `ConfigStore` (déjà
  injecté pour `config.get`, `AdminDispatcher.swift:215-216`).
- CLI miroir `iris config path` : **optionnel/hors-scope** (le RPC sert le « Reveal »
  de l'app ; à ajouter seulement si trivial et utile au smoke).

## 6. Flux de données

- **Affichage** : bootstrap `loadConfig()` + `refreshCATrust()` ; ré-exécutés à
  `.onAppear` de `SettingsTab`. Les `GroupBox` lisent `model.config` / `model.caTrusted`.
- **Édition (D1)** : contrôle modifié → `model.setConfig([update])` → re-fetch →
  feedback « applied » transitoire. Le daemon **valide** (`config.set` rejette
  l'invalide, fichier inchangé, `ConfigStore.applyUpdates`) ; une erreur RPC remonte
  via `RPCError.userMessage` (bandeau, comme 6.2).
- **CA** : Install/Uninstall → `caExportPath()` → `CATrustStore.install/uninstall`
  (shell-out `/usr/bin/security`, panneau d'auth) → `refreshCATrust()`.
- **Reveal/Reload** : Reveal → `configPath()` → `NSWorkspace.shared
  .activateFileViewerSelecting([URL(fileURLWithPath: path)])` ; Reload →
  `reloadConfig()` puis re-fetch.

## 7. Gestion d'erreur

- **`config.set` invalide / daemon down** → bandeau via `RPCError.userMessage` ;
  aucune valeur de secret (ce fichier n'en contient pas) ni dump de hosts inutile
  (CLAUDE.md §6.1).
- **CA install/uninstall échoue** (auth annulée par l'utilisateur, `security`
  exit ≠ 0 → `CAError.trustCommandFailed`) → message d'erreur clair, `refreshCATrust()`
  reflète l'état réel (reste « not trusted »). Pas de crash, pas d'état optimiste figé.
- **`config.path` indisponible** (daemon down) → Reveal désactivé / erreur silencieuse
  loggée, pas de bandeau bloquant.

## 8. Tests

`swift test` (cœur, déterministe) :
- `AppModel.setConfig` : mapping contrôle → `[ConfigSetParams.Update]` correct ;
  re-fetch après set ; erreur RPC propagée (mock `AdminCalling`).
- `AppModel.installCA()` : appelle `caExportPath` **puis** `caInstaller.install(path)`
  (mock `CATrustInstalling`) ; idempotent si déjà trusted (no-op, pas d'appel
  installer). `uninstallCA()` symétrique.
- `AppModel.refreshCATrust()` : met `caTrusted` depuis `isCATrusted()`.
- Bootstrap : liste de calls attendue mise à jour (`+fetchConfig`, `+isCATrusted`).
- `ConfigStore.filePath` / RPC `config.path` : round-trip (chemin résolu == chemin
  du store).

**Smoke poste** (Rule 4 — la vue + le panneau d'auth ne sont pas CI-testables) :
- L'onglet affiche les vraies valeurs ; modifier la politique exfil → vérifiée par
  `iris config get` + persistée + backup créé.
- `max_subs/min` et `backups.max_count` modifiés → reflétés.
- Install CA → panneau d'auth → statut passe « Trusted » → `iris ca is-trusted` OK ;
  Uninstall → « Not trusted ». Idempotence (re-Install no-op).
- Reveal ouvre le Finder sur `config.json` ; Reload re-lit après édition manuelle.

**Gate CI xcodebuild macos-15** : l'app compile → le gate s'applique. Le build local
(toolchain en avance) **n'est pas l'oracle** (leçon durable) → pousser et vérifier le
CI avant merge.

## 9. Critères de réussite (DoD 6.3b)

1. `swift build` + `swift test` + `swift-format` verts ; **CI verte** (gate xcodebuild macos-15).
2. 6e onglet `Settings` visible, sélectionnable, affiche les vraies valeurs de `config.get`.
3. Édition d'un champ hot (`on_exfil_attempt`, `max_subs/min`, `backups.max_count`) →
   appliquée immédiatement + persistée (vérifiable via `iris config get`) + backup créé.
4. Install CA depuis l'app → panneau d'auth → statut « Trusted » (corroboré par
   `iris ca is-trusted`) ; Uninstall → « Not trusted » ; re-Install idempotent.
5. Reveal ouvre le Finder sur le `config.json` réel ; Reload fonctionne.
6. Champs structurels en lecture seule (non éditables) ; aucune bannière « restart »
   (jamais déclenchée).
7. Smoke poste : substitution `claude` inchangée après l'ajout de l'onglet.

## 10. Hors-scope

- Purge Keychain / suppression des secrets en masse (les secrets restent supprimables
  un par un dans Secrets).
- Retrait LaunchAgent / auto-start (→ 6.4 / Phase 7).
- Édition des champs structurels `broker.*` (read-only en 6.3b) ; câblage `log_level`
  backend (→ Phase 5.3.1.x).
- CLI `iris config path` (optionnel ; ajouté seulement si trivial).
- Bannière « redémarrage requis » (sans objet : aucun champ éditable n'est structurel).
